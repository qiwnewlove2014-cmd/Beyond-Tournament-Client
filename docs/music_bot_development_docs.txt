====================================================================
เอกสารบันทึกการพัฒนาระบบ Music Bot (YouTube Real-time Streaming)
====================================================================

1. จุดประสงค์ของระบบ (Purpose)
--------------------------------------------------
ระบบ Music Bot ถูกสร้างขึ้นมาเพื่อยกระดับประสบการณ์ของผู้เล่นในเกม ให้สามารถค้นหาเพลงจาก YouTube และเปิดฟังร่วมกันในขณะเล่นเกมได้แบบ Real-time โดยไม่ต้องพับหน้าจอออกไปเปิดเบราว์เซอร์ และที่สำคัญคือต้องผสมผสาน (Sync) เข้ากับสภาพแวดล้อมและเสียงสะท้อน (Reverb) ของแผนที่ในเกมได้อย่างสมจริง เพื่อให้รู้สึกเหมือนเพลงนั้นถูกเปิดขึ้นมาในโลกของเกมจริงๆ (เสมือนอยู่ในระดับเดียวกับระบบของเกม Sandbox ฟอร์มยักษ์อย่าง Gmod หรือ VRChat)

2. ระดับความยากในการพัฒนา (Difficulty Level)
--------------------------------------------------
ระดับความยาก: 7-8 / 10 (Advanced / ยากมาก)

การพัฒนาฟีเจอร์นี้จัดว่าอยู่ในระดับ "ยากมาก" สำหรับเกมอินดี้ทั่วไป เพราะมันไม่ใช่แค่การเขียนคำสั่งให้เล่นไฟล์ MP3 ที่แถมมากับเกม (ซึ่งใช้โค้ดแค่บรรทัดเดียว) แต่นี่คือการเขียน "Audio Pipeline" ใหม่ทั้งหมดขึ้นมาเอง ซึ่งต้องใช้ความรู้หลายด้าน:
- Low-Level Audio Programming: การย่อยข้อมูลเสียงดิบ (Raw PCM) ที่เพิ่งดาวน์โหลดมาสดๆ ให้เป็นก้อน (Chunk) เล็กๆ แล้วป้อนเข้าการ์ดเสียงผ่านระบบ OpenAL 
- Multi-threading: การรันการดาวน์โหลดและการเล่นเสียงอยู่เบื้องหลัง (Background Thread) เพื่อไม่ให้ตัวเกมหลักกระตุกหรือค้าง
- Memory Management: การบริหารจัดการหน่วยความจำ (RAM) อย่างเข้มงวด หากปล่อยปละละเลยแรมจะบวมและเกมจะแครชทันที
- 3D Audio Routing: การเชื่อมโยงข้อมูลเสียง 2D ให้เข้ากับระบบสภาพแวดล้อม 3D ของเกม

3. อุปสรรค บั๊ก และวิธีการแก้ไข (Obstacles & Bug Fixes)
--------------------------------------------------
ระหว่างการพัฒนา เราเจออุปสรรคทางเทคนิคระดับลึกหลายอย่าง ซึ่งเราได้ทำการแก้ไขจนระบบเสถียร 100% ดังนี้:

[บั๊กที่ 1] ปัญหาเกมค้างเมื่อพิมพ์ค้นหาเพลง
- อาการ: หลังจากกดเสิร์ช YouTube ผู้เล่นไม่สามารถขยับตัวหรือกดปุ่มอะไรในเกมได้เลย เพราะตัวเกมถูกบล็อก
- วิธีแก้: เราพบว่าการสร้าง Input Menu ไปขัดขวางการรับส่ง Event ของเกมหลัก จึงทำการเปลี่ยนให้คำสั่งค้นหาเพลงทำงานบน Background Thread (do_search) และไม่สั่งหยุดเพลงเดิมจนกว่าผู้เล่นจะเลือกเพลงใหม่สำเร็จ ทำให้ผู้เล่นยังคงยิงปืนและเล่นเกมต่อได้ระหว่างที่พิมพ์ชื่อเพลง

[บั๊กที่ 2] ปัญหาหน่วยความจำเต็มและเสียงกระตุก (Memory Leak & Stuttering)
- อาการ: ในเวอร์ชั่นแรก บอทจะสร้าง Buffer ก้อนใหม่ขึ้นมาเรื่อยๆ เพื่อยัดข้อมูลเสียงลงไป ทำให้เมื่อเล่นเพลงไปสักพัก แรมจะเต็ม และเสียงจะเริ่มกระตุก (Buffer Underrun)
- วิธีแก้: เราออกแบบระบบ "Buffer Pool" (การรีไซเคิล) โดยสร้าง Buffer ไว้ล่วงหน้า 16 ก้อน เมื่อก้อนไหนเล่นเสียงเสร็จแล้ว ระบบจะดึงก้อนนั้นกลับมาล้างข้อมูลและใส่ข้อมูลเสียงใหม่เข้าไปแทน ทำให้ระบบกินแรมน้อยลงอย่างมหาศาลและมีความเสถียรสูง

[บั๊กที่ 3] เพลงตายเมื่อกดหยุดชั่วคราว (Pause Stream Death)
- อาการ: เมื่อผู้เล่นกด Pause Thread ที่คอยอ่านข้อมูลจาก YouTube จะหยุดทำงานตามไปด้วย ส่งผลให้ท่อส่งข้อมูล (Pipe) เต็ม และ YouTube จะตัดการเชื่อมต่อ (Timeout) ทำให้เมื่อกด Resume เพลงจะดังแป๊บเดียวแล้วหยุดพร้อมขึ้นข้อความ "Track finished" ทันที
- วิธีแก้: ปรับปรุงตรรกะใหม่ โดยกำหนดให้ Thread ต้อง "ดูดข้อมูลเสียงจาก ffmpeg ต่อไปเรื่อยๆ ห้ามหยุดเด็ดขาด" แม้ผู้เล่นจะกด Pause ก็ตาม โดยนำข้อมูลเสียงที่ดูดมานั้นไปพักไว้ในหน่วยความจำชั่วคราว (collections.deque) แทน เมื่อผู้เล่นกด Resume ก็แค่เอาข้อมูลที่พักไว้มาเล่นต่อ ทำให้ YouTube ไม่ตัดการเชื่อมต่อ

[บั๊กที่ 4] เสียงเพลงดูแห้ง ไม่เข้ากับบรรยากาศในเกม (Environmental Audio Mismatch)
- อาการ: เพลงดังแบบสเตอริโอชัดเจนมาก ซึ่งดีสำหรับหูฟัง แต่อารมณ์มันขัดกับสภาพแวดล้อมของแผนที่ เช่น เดินอยู่ในถ้ำ แต่เสียงเพลงกลับไม่มีเสียงสะท้อน (Reverb) เหมือนเสียงเดินหรือเสียงปืน
- วิธีแก้: เราใช้เทคนิค EFX Reverb Send โดยให้บอทคำนวณตำแหน่งพิกัด (X, Y, Z) ของผู้เล่นแบบเฟรมต่อเฟรม (Real-time) หากผู้เล่นเดินเข้าสู่พื้นที่ที่มี Reverb (เช่น โกดัง หรือถ้ำ) บอทจะนำเสียงเพลงส่งเข้าสู่ท่อสะท้อนเสียงของแผนที่นั้นๆ ทันที ทำให้ได้ผลลัพธ์เป็น "Direct + Reverb" คือเสียงยังคมชัดระดับสเตอริโอ แต่แฝงไปด้วยความก้องกังวานตามบรรยากาศของโลกในเกมจริงๆ

4. บทสรุป (Conclusion)
--------------------------------------------------
ระบบ Music Bot ของ Final Hour Client ตัวนี้ ถือเป็นความสำเร็จทางสถาปัตยกรรมซอฟต์แวร์ที่น่าภูมิใจ การก้าวข้ามข้อจำกัดของ Engine และจัดการกับระบบเสียงระดับ Low-level ได้สำเร็จ ทำให้ตัวเกมมีฟีเจอร์ระดับเดียวกับเกม Sandbox ยุคใหม่ ซึ่งจะช่วยสร้างสีสันและความสนุกสนานให้กับการรวมกลุ่มของผู้เล่นในเกมได้อย่างยอดเยี่ยม
